# SellerRevenueSummaryDto

Represents an aggregated financial summary for a seller within the marketplace, consolidating gross and net revenue, platform fees, profit margins, pending payouts, and a month-by-month breakdown of revenue performance. This DTO is typically returned by reporting or analytics services to provide a comprehensive view of a seller’s earnings over a given period.

## API

### SellerId
`public Guid SellerId`

The unique identifier of the seller to whom this revenue summary belongs. This value correlates with the seller’s identity across the marketplace platform and is immutable once set.

### TotalGrossRevenue
`public decimal TotalGrossRevenue`

The total revenue generated by the seller before any deductions, such as platform fees, refunds, or chargebacks. This value is expressed in the marketplace’s base currency and represents the sum of all successful transactions within the summary period.

### TotalNetRevenue
`public decimal TotalNetRevenue`

The revenue remaining after all platform fees and other mandatory deductions have been subtracted from the gross revenue. This is the amount the seller effectively earned and is typically the basis for payout calculations.

### TotalPlatformFees
`public decimal TotalPlatformFees`

The cumulative amount deducted by the platform as fees across all transactions in the summary period. This includes listing fees, transaction commissions, and any other service charges applied by the marketplace.

### PlatformFeePercentage
`public double PlatformFeePercentage`

The effective platform fee rate expressed as a percentage, calculated as `(TotalPlatformFees / TotalGrossRevenue) * 100`. If `TotalGrossRevenue` is zero, this property may be zero or NaN depending on implementation; consumers should guard against division-by-zero scenarios.

### ProfitMarginPercentage
`public double ProfitMarginPercentage`

The seller’s profit margin expressed as a percentage, calculated as `(TotalNetRevenue / TotalGrossRevenue) * 100`. This reflects the proportion of gross revenue retained by the seller after platform fees. As with `PlatformFeePercentage`, a zero gross revenue may produce a zero or NaN value.

### PendingPayout
`public decimal PendingPayout`

The total amount currently awaiting disbursement to the seller. This includes net revenue that has been cleared but not yet transferred, and excludes amounts already paid out or still under hold or review.

### MonthlyBreakdown
`public List<MonthlyRevenueDto> MonthlyBreakdown`

A chronological list of revenue summaries grouped by month, each represented by a `MonthlyRevenueDto`. This list enables time-series analysis and is ordered from earliest to latest month within the summary period. The list may be empty if no revenue was generated.

## Usage

### Retrieving and Displaying a Seller Revenue Summary
```csharp
public async Task DisplaySellerSummaryAsync(Guid sellerId)
{
    var summary = await _revenueService.GetSellerRevenueSummaryAsync(sellerId);

    Console.WriteLine($"Seller: {summary.SellerId}");
    Console.WriteLine($"Gross Revenue: {summary.TotalGrossRevenue:C}");
    Console.WriteLine($"Net Revenue: {summary.TotalNetRevenue:C}");
    Console.WriteLine($"Platform Fees: {summary.TotalPlatformFees:C} ({summary.PlatformFeePercentage:F2}%)");
    Console.WriteLine($"Profit Margin: {summary.ProfitMarginPercentage:F2}%");
    Console.WriteLine($"Pending Payout: {summary.PendingPayout:C}");

    foreach (var month in summary.MonthlyBreakdown)
    {
        Console.WriteLine($"  {month.Year}-{month.Month:D2}: Net {month.NetRevenue:C}");
    }
}
```

### Calculating Year-to-Date Performance from Monthly Breakdown
```csharp
public decimal CalculateYearToDateNetRevenue(SellerRevenueSummaryDto summary, int year)
{
    return summary.MonthlyBreakdown
        .Where(m => m.Year == year)
        .Sum(m => m.NetRevenue);
}
```

## Notes

- All monetary values (`TotalGrossRevenue`, `TotalNetRevenue`, `TotalPlatformFees`, `PendingPayout`) use `decimal` to preserve precision and avoid floating-point rounding errors common in financial calculations.
- `PlatformFeePercentage` and `ProfitMarginPercentage` are `double` and may produce `NaN` or `PositiveInfinity` if `TotalGrossRevenue` is zero. Always check for a non-zero denominator before performing further arithmetic with these percentages.
- `MonthlyBreakdown` is a `List<MonthlyRevenueDto>` and is not thread-safe for concurrent writes. If multiple threads enumerate or modify the list simultaneously, external synchronization (e.g., `lock` or `ImmutableList`) is required.
- This DTO is a plain data container and does not implement `INotifyPropertyChanged` or any mutation tracking; instances are intended to be consumed as snapshots and should be treated as effectively immutable after construction.
- The `PendingPayout` field reflects the state at the time the summary was generated and may change independently of the other summary values as payouts are processed. Do not assume it equals `TotalNetRevenue` minus previously paid amounts without consulting the payout service.
