# SellerDashboardDto

Data transfer object representing key metrics and details for a seller's dashboard in the marketplace engine. Aggregates live sales data, listing status, financial summaries, and recent activity to provide a consolidated view for seller analytics and reporting.

## API

### `SellerId`
Unique identifier for the seller. Used to associate dashboard data with a specific seller account.

### `SellerName`
Display name of the seller as shown to buyers. Used in UI and communications.

### `ActiveListings`
Current count of active product listings managed by the seller. Reflects inventory currently available for purchase.

### `TotalListings`
Total historical count of product listings created by the seller, including inactive or sold items.

### `TotalSales`
Cumulative number of completed sales transactions attributed to the seller.

### `TotalRevenue`
Sum of all revenue generated by the seller, including fees and payouts. Expressed in the system‚Äôs base currency.

### `PendingPayout`
Amount currently pending disbursement to the seller‚Äôs account. Includes unsettled earnings from recent transactions.

### `AverageRating`
Arithmetic mean of all buyer ratings received by the seller, rounded to two decimal places. Null if no reviews exist.

### `TotalReviews`
Total number of reviews submitted by buyers for the seller‚Äôs transactions.

### `UnreadMessages`
Count of unread messages in the seller‚Äôs inbox. Used to indicate communication backlog.

### `LastActivityAt`
Timestamp of the seller‚Äôs most recent interaction (e.g., login, message sent, listing updated). Null if no activity recorded.

### `TotalGrossRevenue`
Total revenue before any platform fees or deductions. Represents gross earnings from all sales.

### `TotalPlatformFees`
Sum of all fees deducted by the platform from the seller‚Äôs transactions.

### `TotalNetRevenue`
Net revenue after platform fees and refunds. Calculated as `TotalGrossRevenue - TotalPlatformFees`.

### `PendingPayout`
Amount awaiting payout to the seller‚Äôs financial account. Excludes any pending disputes or chargebacks.

### `MonthlyBreakdown`
List of monthly revenue summaries for the current and prior months. Each entry contains granular revenue data by month.

### `Year`
Calendar year for which the dashboard data is aggregated. Used to scope time-based metrics.

### `Month`
Calendar month (1‚Äì12) for which the dashboard data is aggregated. Used to scope time-based metrics.

### `GrossRevenue`
Total gross revenue for the specified month and year. Reflects pre-fee earnings.
