# WebhookEvent

Represents an incoming webhook payload received by the marketplace engine, encapsulating event metadata, payload data, and verification details.

## API

### `public string EventType`
Gets the type of the webhook event (e.g., `payment.processed`, `shipping.updated`). This value is used to route the event to the appropriate handler.

### `public DateTime Timestamp`
Gets the UTC timestamp when the webhook was generated by the sender. Used for ordering and validation of event freshness.

### `public Dictionary<string, object?> Data`
Gets the payload data associated with the webhook event. The dictionary contains key-value pairs representing the event-specific data. May be `null` if no data is provided.

### `public string? Signature`
Gets the cryptographic signature provided by the sender to verify the authenticity and integrity of the webhook payload. May be `null` if no signature was included.

### `public WebhookService`
Gets the service instance associated with this webhook event, used for registration and processing.

### `public void RegisterHandler(WebhookHandler handler)`
Registers a handler for a specific event type. The handler will be invoked when an event of the registered type is processed.

- **Parameters**:
  - `handler`: The `WebhookHandler` delegate to register.
- **Throws**:
  - `ArgumentNullException`: If `handler` is `null`.

### `public async Task ProcessWebhookAsync()`
Processes the webhook event by validating its signature (if present), routing it to the appropriate handler based on `EventType`, and invoking the registered handler. This method is asynchronous to accommodate I/O operations during validation and handler execution.

- **Throws**:
  - `InvalidOperationException`: If no handler is registered for the event type.
  - `SecurityException`: If the signature is present but invalid.

### `public string GenerateSignature(string payload, string secret)`
Generates a cryptographic signature for the provided payload using the specified secret. The signature is typically used by senders to sign outgoing webhooks.

- **Parameters**:
  - `payload`: The raw payload string to sign.
  - `secret`: The secret key used for signing.
- **Returns**: A hexadecimal string representing the HMAC-SHA256 signature of the payload.
- **Throws**:
  - `ArgumentNullException`: If `payload` or `secret` is `null`.

### `public PaymentWebhookHandler`
Gets the handler specifically for payment-related webhook events. This handler is invoked when `EventType` matches a payment event.

### `public async Task HandleAsync(PaymentWebhookEventArgs args)`
Handles a payment-related webhook event asynchronously. This method processes the event data, updates the system state, and performs any necessary actions (e.g., updating order status, sending confirmations).

- **Parameters**:
  - `args`: The event arguments containing the webhook data and context.
- **Throws**:
  - `ArgumentNullException`: If `args` is `null`.
  - `InvalidOperationException`: If the event data is malformed or required fields are missing.

### `public ShippingWebhookHandler`
Gets the handler specifically for shipping-related webhook events. This handler is invoked when `EventType` matches a shipping event.

### `public async Task HandleAsync(ShippingWebhookEventArgs args)`
Handles a shipping-related webhook event asynchronously. This method processes the event data, updates shipment records, and triggers downstream processes (e.g., notifying customers, updating inventory).

- **Parameters**:
  - `args`: The event arguments containing the webhook data and context.
- **Throws**:
  - `ArgumentNullException`: If `args` is `null`.
  - `InvalidOperationException`: If the event data is malformed or required fields are missing.
